  ☐ 1.4 Error Handling

MODULE 6: TRANSACTION MANAGEMENT
================================================================================

Người phụ trách: [CẦN PHÂN CÔNG]

Endpoints:
  - POST /api/transactions/purchase (Tạo giao dịch mua - ADMIN)
  - POST /api/transactions/sell (Tạo giao dịch bán - USER)
  - POST /api/transactions/return (Tạo giao dịch hoàn trả - USER)
  - GET /api/transactions/all (Lấy tất cả giao dịch + phân trang)
  - GET /api/transactions/{id} (Lấy giao dịch theo ID)
  - GET /api/transactions/by-month-year (Lấy giao dịch theo tháng/năm)
  - PUT /api/transactions/update/{transactionId} (Cập nhật trạng thái)

VẤN ĐỀ CẦN GIẢI QUYẾT:
  ☐ 6.1 Transaction Status Management
       - Status: PENDING, COMPLETED, CANCELLED, FAILED
       - Định nghĩa rõ workflow trạng thái
       - Validation: chỉ cập nhật status nếu hợp lệ

  ☐ 6.2 Stock UpdateTypes & Status
       - Types: PURCHASE (mua), SELL (bán), RETURN (hoàn trả)
       - Status: PENDING, COMPLETED, CANCELLED
       - Define state transitions (PENDING -> COMPLETED -> CANCELLED?)

  ☐ 6.2 Stock Update Logic (CRITICAL)
       - PURCHASE: stock += quantity (add stock)
       - SELL: stock -= quantity (reduce stock, check enough?)
       - RETURN: stock += quantity (add stock back)
       - Handle transaction rollback nếu stock update fail

  ☐ 6.3 Validation & Business Rules
       - Quantity > 0
       - Product tồn tại
       - Supplier tồn tại (cho PURCHASE/RETURN)
       - Check stock >= quantity trước khi SELL
       - Calculate totalPrice = quantity * price tại thời điểm transaction

  ☐ 6.4 Transaction Creation
       - Create transaction theo type (purchase/sell/return)
       - Update stock tự động
       - Set status = COMPLETED (hoặc PENDING?)
       - Save transaction to database

  ☐ 6.5 Filter & Pagination
       - Pagination: page, size (supported)
       - Filter by status, trạng thái (nếu có)
       - Sort by date descending

  ☐ 6.6 Get Transactions by Month/Year
       - Query transactions trong tháng/năm cụ thể
       - Filter TransactionType nếu cần

  ☐ 6.7 Update Transaction Status
       - Update status của transaction
       - Validate state transition hợp lệ
       - Log status changes

CODING STANDARDS & BEST PRACTICES
================================================================================

1. Exception Handling
   - Extend custom exceptions từ RuntimeException
   - Use GlobalExceptionHandler cho consistent error response
   - Return proper HTTP status codes

2. Response Format
   - All endpoints return Response object: { status, message, data }
   - Success: status = 200, message = "success", data = object
   - Error: status = code, message = error message, data = null

3. Validation
   - Use @Valid @NotBlank @Email @Positive @Min on DTO fields
   - Return 400 Bad Request với clear error message
   - Validate foreign keys (product, category, supplier tồn tại)

4. Authorization
   - Use @PreAuthorize("hasAuthority('ADMIN')") cho admin operations
   - Check current user ownership trước khi update/delete profile
   - Return 403 Forbidden nếu không có quyền

5. Transactions
   - Use @Transactional trên methods thay đổi stock
   - Rollback automatic nếu exception
   - NO manual transaction management

6. DTOs
   - Use ModelMapper để map giữa DTO <-> Entity
   - Never expose password, sensitive data trong response DTO
   - Use separate request/response DTOs nếu khác nhau
